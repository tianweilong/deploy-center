# AGENTS

## 语言与文案规范

- 默认使用中文：Issue、PR 描述、评审评论、自动化代理回复、设计文档与发布说明均优先中文。
- 代码内可见文案优先中文：注释、CLI 帮助文案、交互提示、错误信息、日志输出、脚本提示信息、配置模板注释。
- 测试相关文案优先中文：测试名称说明、断言失败提示、表驱动用例描述、测试数据注释。
- 外部协议或第三方约束保持原文：如 API 字段名、协议常量、第三方错误码；必要时可使用“英文原文 + 中文说明”。
- 无明确要求时遵循“能中文尽量中文”，仅在兼容性或行业标准要求下使用英文。

## 兼容性边界

- 不修改环境变量名、API 字段名、JSON/YAML 键名、协议字段名、镜像仓库名、仓库路径和命令名。
- 修改文案时优先保持自动化行为不变，避免影响脚本、工作流和外部集成。
