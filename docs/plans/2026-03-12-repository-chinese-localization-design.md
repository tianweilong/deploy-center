# 仓库中文化设计

## 背景

`deploy-center` 当前仓库中仍存在较多英文的人类可读文案，包括仓库说明、部署文档、Webhook 说明、脚本提示与测试失败信息。为统一团队协作体验，需要将仓库主要语言调整为中文，同时保持现有脚本、配置和外部协议兼容。

## 目标

- 将仓库中的主要自然语言内容调整为中文。
- 新增根级 `AGENTS.md`，明确仓库默认使用中文的长期约束。
- 保留外部协议、第三方约束和机器契约字段不变，避免破坏自动化流程。

## 范围

本次调整覆盖以下内容：

- 仓库说明文档：`README.md`
- 设计与发布文档：`docs/**/*.md`
- Webhook 文档：`agents/webhook/**/*.md`
- 环境说明：`environments/**/README.md`
- 面向人的脚本提示：`scripts/*.sh`、`scripts/*.rb`
- 测试中的说明性文案与断言失败提示：`tests/*.sh`
- 示例配置注释：`.env.example`

## 非目标

以下内容不在本次中文化范围内：

- JSON/YAML 键名
- 环境变量名
- API 字段名、协议字段名、第三方错误码
- 镜像仓库名、仓库路径、命令名
- GitHub Actions 的表达式、键名与内建字段

## 设计原则

1. 默认中文：所有仓库内面向人的新增或修改文案优先使用中文。
2. 兼容优先：机器消费的标识符、字段名和协议约束保持原文。
3. 语义翻译：按语义逐文件翻译，不做危险的全局替换。
4. 可验证：通过测试与全文扫描双重验证中文化结果。

## 实施方式

### 文档层

- 将顶层说明、架构文档、发布文档、历史计划文档统一为中文表达。
- 在涉及外部接口时，采用“英文原文 + 中文说明”或保留关键英文标识。

### 脚本层

- 将脚本中的 `echo`、`abort`、参数错误提示改为中文。
- 不调整脚本入参名、环境变量名与配置路径。

### 测试层

- 先修改测试中的期望与提示文案，形成失败测试。
- 再修改实现文件，使测试恢复为绿色。
- 新增一个面向仓库中文化约束的测试，集中检查关键文档与脚本文案是否已转为中文。

### 规则固化

- 新增根级 `AGENTS.md`，写明默认中文规范与例外规则。

## 风险与缓解

- 风险：误改协议键名或配置字段导致自动化失败。
  - 缓解：仅翻译自然语言，不改结构键名。
- 风险：历史文档中的命令或标识被误翻译。
  - 缓解：命令、环境变量、镜像名一律保留原文。
- 风险：测试仍引用旧英文文案。
  - 缓解：先写/改失败测试，再改实现。

## 验证策略

- 运行现有 `tests/*.sh`，并让与中文化有关的断言全部通过。
- 使用 `rg` 扫描残留英文说明文本，确认保留项仅限协议、标识符与第三方约束。

## 预期产出

- 新增：根级 `AGENTS.md`
- 新增：中文化设计文档与实施计划文档
- 修改：文档、脚本、测试及示例配置中的主要自然语言内容
